Git's Dictionary

Some git's commands :

  • git diff : Show changes between commits. Ex : git diff
  • gitk --all : Show grapgical interface. Ex : gitk --all
  • git branch : Create a new branch Ex : git branch branch_name
  • git tag : Tag a specific commit. Ex : git tag -a v1.0 -m 'tag'
  • git cat-file : View the type of an object. Ex : git cat-file -t
  • git add : Add files changes in the working directory. Ex : git add
  • git checkout : changes working branch. Ex : git checkout branch_name
  • git show : Show informations about the project. Ex : git show commit
  • git status : Obtain a summary of which files have changes. Ex : git status
  • git commit -m : Create a new version of the projet. Ex : git commit -a -m 'comment'
  • git ckeckout -b : Create a branch and switch on it. Ex : git checkout -b new_branch
  • git init : Initialize a git repository. Ex : cd /home/user/my_new_git_folder/ git init
  • git merge : Put branch's files on a current branch (master). Ex : git merge branch_name
  • git checkout : Check/switche out/to a different branch. Ex : git checkout new_branch_name
  • git commit --amend -m : Change a commit's comment. Ex : git commit --amend -m "New_Comment"
  • git reset : Reset index and working directory to your last commit. Ex : git reset --hard HEAD
